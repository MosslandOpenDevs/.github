# Contributing

These are the default contribution guidelines for repositories in the
MosslandOpenDevs organization. Check the repository you are contributing to
for its own `CONTRIBUTING.md` first; a repository-level document takes
precedence over this one.

## Before you start

- Open an Issue before starting a large or structural pull request, so the
  approach can be discussed before the work is done.
- Small, self-evident fixes (typos, broken links, obvious one-line bugs) may
  go straight to a pull request.

## Pull requests

- Keep each pull request scoped to one change. Split unrelated changes into
  separate pull requests.
- Describe what you changed and why in the pull request description.
- Describe the verification you performed: which commands, tests, or checks
  you ran and what you observed. "Not verified" is an acceptable answer;
  silence is not.

## What not to include

- Do not include secrets, credentials, tokens, or personal data in issues,
  pull requests, or commits — including in logs, screenshots, and test
  fixtures.
- Do not report suspected vulnerabilities in public issues or pull requests.
  Use the private reporting path described in [SECURITY.md](SECURITY.md).
